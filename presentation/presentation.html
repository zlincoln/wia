Masonry Plugin Demo

For Tiled Grid Layouts

Installation

With Bower via command line

bower install masonry
bower install imagesloaded

Don't have Bower?

Install Node, then via command line:

sudo npm install -g bower
bower install masonry
bower install imagesloaded

Add the following scripts </body>

<script src="bower_components/masonry/dist/masonry.pkgd.min.js"></script>
<script src="bower_components/imagesloaded/imagesloaded.pkgd.min.js"></script>

In this demo, I'm assuming you're using it with jQuery, though you can get by without it.

Initialization

The most basic initilization is as simple as

$container.masonry();

Doesn't need to be contained within document.ready

Masonry takes each item within the container and repositions it absolutely based upon width and height. This can cause issues when images are still loading and the element hasn't reached its full height.

The solution is to use imagesLoaded, which runs a function you specify once all child elements have been fully rendered.

imagesLoaded($container, function(){
						$container.masonry();
});

Dynamically adding and removing elements

How would you add elements after the inital layout? Steps are as follows:

  1. Append new elements to the DOM
  2. Add them to the masonry registry
    $container.masonry('addItems', elementArray)
  3. Trigger masonry relayout
    $container.masonry();

Removing elements is easier:

$container.masonry('remove', elementArray);

Elements are removed from registry and DOM

Making it pretty

By default, masonry tiles appear stacked and then fan out to their positions like cards. However, using an emmitted event called 'layoutComplete' we can fade them in when in final position.

Syntax for catching masonry events is similar to jquery.

$container.masonry('on', 'layoutComplete', callback);

In this case, the callback function would add a 'loaded' class to the child elements. In my css, those elements have an opacity of zero until the loaded class had been attached.

Be sure to check out the documentation for all options, methods, and events!
